Testing Concepts

  1. What is the purpose of a structured testing environment?
    1. Having a structured testing environment is about how the system does the tasks instead of how it works. By looking at the structure of the code and fully understanding what each part is supposed to do it helps catch errors, verify the flow, verify expected outputs and verify that each part of the code works as expected.
    2. http://www.softwaretestingclass.com/what-is-structural-testing/

 

  1. Identify and describe the main types of tests performed on software, including web applications?
    1. Web app testing is comprised of:
      1. Functional Testing: which checks for invalid redirects, validation, workflow, data integrity. This test focuses on the functionality of the app.
      2. Usability testing: Checks how easy/hard the application is to use.
      3. Interface testing: checks the interface and dataflow from one system to another.
      4. Compatibility testing: Checks for browser, operating system and device compatibility
      5. Performance testing: Checks for server response time and load conditions. A subset of performance testing is, Load testing, stress testing, soak testing (checks system parameters under continuous expected load), and spike testing (checks whether the system will be able to sustain a work load in a sudden increase of users).
      6. Security testing:  checks to make sure the application is secured for the web against data theft.
      7. https://www.tutorialspoint.com/software_testing_dictionary/web_application_testing.htm
    2. Software testing is comprised of:
      1. Sanity: tests for basic functionality
      2. Smoke: tests for functionality of applications core features
      3. Functional: thoroughly tests the functionality of a feature
      4. Regression: tests older features to ensure changes haven’t had an impact
      5. Integration: tests the different parts of an app to make sure they are working in the way you expect.
      6. Acceptance: testing requirements for the client
      7. Unit: tests specific part of code to make sure it works correctly.

 

  1. Which of these tests are most applicable for automated testing and web applications?
    1. Automated testing is best used with regression tests. Web applications on the other hand go well with functional, usability, interface, compatibility, performance, and security testing.
